markdown\n# KOH‑FEE‑BREW QR‑order site\n\n## What you get\n- A mobile‑first static website (HTML + CSS + JS) that lets a guest:\n 1. Scan a QR code → open the page.\n 2. Enter their table number.\n 3. Browse a full menu (images, veg/non‑veg icons, price).\n 4. Add items to a cart, optionally write a note per item.\n 5. Click **Proceed to Checkout** → friendly confirmation toast.\n\nNo backend is required for the basic flow; the order data is logged to the browser console.\n\n---\n## Quick local test\n1. Create a folder on your computer, e.g. `kohfeebrew-qr`.\n2. Inside that folder create **five** files with the exact names above and paste the corresponding content.\n3. Add an `images/` sub‑folder and drop any placeholder pictures (or the real photos) using the filenames referenced in `menu-data.js`. If you don’t have images yet, you can copy a single generic placeholder image and name it `placeholder.jpg`, then change every `img` value to `images/placeholder.jpg`.\n4. Open a terminal in the folder and run a tiny static server:\n bash\n # Python 3 (comes with most OSes)\n python -m http.server 8000\n \n5. Open a browser on your phone (or on the same machine) and go to `http://localhost:8000`. The site should load, let you enter a table number, and you can start ordering.\n\n---\n## Deploying for free\n### Option 1 – GitHub Pages\n1. **Create a new repo** on GitHub (you can use the free tier). Name it e.g. `kohfeebrew-qr`.\n2. **Upload the files**:\n - **Via web UI** → click **Add file → Upload files**, drag‑and‑drop the five files (and the `images/` folder) and commit.\n - **Or via CLI** (if you have Git installed):\n bash\n git init\n git remote add origin https://github.com/your‑user/kohfeebrew-qr.git\n git add .\n git commit -m "Initial site"\n git push -u origin master\n \n3. Go to the repo **Settings → Pages** (or **Code & Automation → Pages**). Choose the `main` (or `master`) branch and the root (`/`). Save.\n4. GitHub will give you a URL like `https://your‑user.github.io/kohfeebrew-qr/`. That is the link you will embed in the QR code.\n\n### Option 2 – Netlify Drop (no account needed)\n1. Zip the whole folder (`kohfeebrew-qr.zip`).\n2. Visit https://app.netlify.com/drop and drag the zip file onto the page.\n3. Netlify will instantly host it and give you a URL (`https://something‑netlify.app`).\n4. Use that URL for the QR.\n\n### Option 3 – Vercel (free tier, quick CLI)\nbash\nnpm i -g vercel # one‑time install\nvercel # run inside the folder, answer prompts\n\nVercel will output a preview URL and, after confirming, a production URL.\n\n---\n## Generating the QR code\n- Open https://qr-code-generator.com/ (or any QR‑generator you like).\n- Paste the final URL you obtained from GitHub Pages / Netlify / Vercel.\n- Choose **static** (no tracking), generate, download the PNG, and print it on stickers or table placards.\n\n---\n## Next steps (optional)\n- **Backend** – replace the `console.log` in `app.js` with a `fetch('/api/orders', …)` call to send the JSON to a simple server (Node/Express, Google Sheets via Apps Script, or a low‑code tool like Zapier).\n- **Live menu sync** – write a small script that scrapes the Google Maps “Menu” page and overwrites `menu-data.js` automatically (run it on a schedule via GitHub Actions or a cheap VPS).\n- **Push notifications** – add a tiny Web‑Push library to notify the guest when the kitchen marks the order ready.\n\nThat’s all you need to get the site up and running without any paid “Emergent Pro” plan. Good luck, and enjoy watching guests order straight from their phones! 🚀\n
